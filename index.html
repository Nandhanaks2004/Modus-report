Section 09

Security Architecture

JWT Authentication

Stateless tokens with configurable expiry (default 24h). Tokens encode user ID and role for efficient authorization.

Password Hashing

PBKDF2-SHA256 via Werkzeug. Plain-text passwords are never stored or logged.

Role-Based Access

@login_required and @admin_required decorators protect all routes. Student tokens cannot access admin endpoints.

Protected Routes

Frontend route guards prevent unauthorized page access before API calls are made.

File Upload Validation

Extension whitelist/blacklist validation. Disallowed extensions: exe, dll, bat, msi, apk, ps1, scr.

CORS Protection

Cross-Origin Resource Sharing configured via Flask-CORS. Restrictable to specific origins in production.

Section 13

Final Product Assessment

Strengths

  • Unified platform eliminates tool fragmentation — one login for LMS, CTF, exams, and labs.
  • Zero-infrastructure Linux lab — browser-based terminal simulator requires no VMs, Docker, or cloud instances.
  • Complete CTF platform with flag validation, XP rewards, and leaderboard integration.
  • Curriculum-aligned content — instructors create challenges that directly match their teaching modules.
  • Modern architecture — React/Flask/SQLite stack with clear separation of concerns.
  • Self-hosted — full institutional control over data and infrastructure.

Unique Features

  • Browser-based Linux Terminal — virtual filesystem with 10+ command handlers, no external infrastructure.
  • Integrated CTF + LMS — challenges embedded directly into the learning flow, not a separate platform.
  • Batch-isolated content — run multiple courses simultaneously with independent curricula.
  • Gamified cybersecurity training — XP, leaderboards, and rankings built into every module.

Market Potential

MODUS addresses a clear gap in the cybersecurity training market. Existing solutions either provide LMS functionality (Moodle, Google Classroom) without practical labs, or provide CTF training (HTB, TryHackMe) without institutional management. MODUS is the only platform that combines both in a self-hosted, curriculum-aligned package.

Target markets include: cybersecurity training institutes, SOC academies, university cybersecurity programs, corporate training departments, and bootcamp providers.

Technical Value

The platform demonstrates production-quality software engineering: RESTful API design, JWT-based security, normalized database schema, component-based UI architecture, service layer pattern, and comprehensive error handling. The Linux terminal simulator represents a particularly strong technical achievement, implementing a virtual filesystem and command interpreter entirely in the browser.
